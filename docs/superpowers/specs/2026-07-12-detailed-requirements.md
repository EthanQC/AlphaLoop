## 0. 系统总览

**一句话**：一个跑在 Mac mini 上的小圈子美股交易辅助平台——每个成员对等接入：自动收报告、盯盘提醒、站内提问做研究、提案自批自复盘，系统记住每个人的策略与判断并越用越懂你；成员间通过策略名片互相看牌路。

### 0.1 产品定位（已拍板）

**小圈子产品**：当前成员 = 你 + mashu（完全对等）；架构上加第三个朋友 = 装 skill + 领账号 + 开长桥模拟盘，配置级操作而非重构。不做公开注册、不做商业化运营。营销含义只有一条：**接入体验要好到朋友愿意装**（30 分钟内从零到收到第一张提醒卡）。

### 0.2 角色与设备

| 角色/设备 | 职责 |
|---|---|
| **成员**（你、mashu，未来更多） | 权限完全对等：各自的模拟盘（提案-审批-复盘）、各自的提醒规则、各自的标的池、各自的策略记忆；互相能看到对方的**公开档**内容与名片 |
| **Mac mini**（平台服务器） | 自动化生产（行情/新闻/报告/提醒/提案）+ 平台（Web 应用）+ 共享服务（查询/发布/研究任务）+ 各成员数据的隔离存储。无人值守、重启自愈 |
| **每人的本地 OpenClaw**（个人工作台） | 装 AlphaLoop 客户端 skill + 个人 token：本地自治跑，私有策略/偏好存本地；需要平台数据时调平台接口；发布策略/论点时按可见性档位推送 |
| **飞书** | 每人单聊：自己的提醒卡、提案审批卡、个人复盘；群：公共通知（报告发布/系统告警）、讨论、@ 机器人对话 |

### 0.3 八类交互全景

| # | 交互 | 发生在哪 |
|---|---|---|
| 1 | 读报告/分析/复盘 | 平台（报告页；个人内容登录后可见） |
| 2 | 读新闻 | 平台（新闻页） |
| 3 | **站内提问做研究** | 平台（首页"开始研究"入口 → 研判页） |
| 4 | 看状态（持仓/规则/提案） | 平台（模拟盘页、首页个人区） |
| 5 | 收通知 | 飞书单聊（个人）/ 群（公共），深链跳平台 |
| 6 | 说一句话（管规则/记判断/要提案） | 飞书对话；本地 OpenClaw |
| 7 | 批提案 | 飞书卡片按钮（各批各的） |
| 8 | 策略记忆与名片 | 采集：飞书+审批卡+本地工作台；呈现：策略页+名片；回流：注入本人的报告/提案/研判 |

### 0.4 立项原则（全文有效）

- **不做实时行情看盘**（券商 App 的事）；平台数据分钟到小时级刷新，每个数字标注数据时间。
- **时间约定**：全站统一显示北京时间；日报在每个美股交易日开盘前（北京时间约 20:00）生成；"提醒流水"指最近一个美股交易时段（北京时间约 21:30–次日凌晨 4/5 点）；提醒配额按美东交易日切界。
- **标的池**：每人各自维护（每人上限 20 只）；平台的新闻抓取与个股分析按**全体成员标的池的并集 + 全体持仓**生产（个股分析是公共资产，谁都能看）。
- **预算与频控**：新闻检索日报 ≤30 次、周报 ≤60 次、深挖每事件 ≤5 轮（周报 ≤8 轮）；报告生成总时长 ≤15 分钟；提醒卡每人每日 ≤30 张；站内研究每人每日 ≤10 次（防预算失控，可调）。
- **机械数据自动填、人只填判断**；**"不确定"是一等公民**；每个结论带证据与置信度。
- **PDF 已退役**；美股节假日无新输入不产出。
- 安全宪法沿用：永不实盘、期权永不自动化、只有下单服务可触达券商、每人模拟盘 ≤10% 预算服务端校验、凭据不入仓不入提示词不入报告、飞书白名单、全中文。

## 1. 平台（Web 应用）

### 1.1 全局设计

**形态**：从"静态站点"升级为 **Web 应用**——静态内容（报告/新闻/历史）+ 动态能力（登录身份、个人内容、站内研究）。访问 `https://reports.<你的域名>`，白名单邮箱验证码登录（登录即身份：平台据此渲染你的个人内容与权限），会话 30 天。

**导航**：固定底部标签栏 5 tab（图标+中文短标签）：

```
┌─────────────────────────────────────┐
│              页面内容                │
├─────────────────────────────────────┤
│ 🏠首页  📄报告  📰新闻  💼模拟盘  🎯策略 │
└─────────────────────────────────────┘
```

"研究"不占 tab——入口是首页顶部的**提问框**（FinSight 模式），研判结果归档进报告页（类型=研判）。

**每页通用元素**：生成时间条 + 新鲜度标签（最新/延迟/部分缺失）+ 降级横幅（数据缺失时变灰对应区块并如实标注，绝不静默）；全中文；页面不向第三方发任何请求；深链规则稳定（`/daily/<date>`、`/stock/<code>`、`/proposal/<id>`、`/research/<id>`、`/member/<who>`）。体验目标：手机已登录状态数秒内可读（不作验收门槛）。

**视觉与主题（已定稿，2026-07-12）**：
- **双主题可切换**：浅色主题=「作战室」bento 风（样张 C），暗色主题=「盯盘室终端」风（样张 A，含顶部持仓行情纸带与状态灯签名元素）；默认跟随系统，页内一键切换并记住选择。
- **涨跌配色：绿涨红跌**（美股习惯）。
- **全端响应式**：移动端单手优先（底部 5 tab、单列/双列卡片流）；**桌面端（≥1024px）左侧导航栏 + 多列 bento 网格**，充分利用宽屏信息密度。两端同一信息架构。

### 1.2 首页（登录后个人化）

```
┌─────────────────────────────────────┐
│ 07-12 周日 · 生成 20:05 · [最新]      │
├─────────────────────────────────────┤
│ 🔍 开始研究                           │
│ ┌─────────────────────────────────┐ │
│ │ 问点什么…如"NVDA 财报前要减仓吗"  │ │
│ └─────────────────────────────────┘ │
│ 最近研判: NVDA 止损位研判(昨天) →     │
├─────────────────────────────────────┤
│ 我的模拟盘  净值1.083 今日+0.6%       │
│ ▁▂▃▅▄▆▇  [对比 mashu →]            │
├─────────────────────────────────────┤
│ ⏳ 我的待办                           │
│ ▸ 提案 P-124 NVDA 买 2 股 (23h)      │
├─────────────────────────────────────┤
│ 🔔 我的提醒（最近交易时段, 3 条）       │
├─────────────────────────────────────┤
│ 📄 今日日报「联储纪要偏鹰…」[阅读→]    │
│    我的个人页：持仓速览+策略对照 [→]   │
├─────────────────────────────────────┤
│ 🎯 我的纪律速览：财报周不加仓(NVDA 7/28)│
└─────────────────────────────────────┘
```

区块自上而下：**开始研究**（提问框+最近研判入口）→ 我的模拟盘概览（净值/今日/sparkline+对比入口）→ 我的待办（未决提案）→ 我的提醒流水 → 今日日报卡（公共报告+我的个人页两个入口）→ 我的纪律速览（匹配规则：临近财报→财报类纪律；接近仓位上限→仓位纪律；周亏接近 3%→熔断预警；无匹配→"已连续遵守 N 天"）。**全站登录后可见**（Access 白名单即门禁），不存在未登录浏览态——简单且更安全。

### 1.3 站内研究（新，v1 就做）

**一句话**：像 FinSight 一样在平台上直接提问，看着系统调研，拿到"结论先行+置信度"的结构化研判——但答案带着**你的**持仓、论点和纪律。

**流程**：

1. **提问**：首页输入框，自然语言（"NVDA 财报前要减仓吗""帮我定 TSLA 的止损位"）。仅研究类问题；操作类（改规则/批提案/记记忆）提示走飞书。
2. **进行中页**：提交后立即得到任务页，**调研过程实时可见**（步骤流：意图解析 → 拉取行情 → 检索新闻 → 读取你的论点与纪律 → 数字校验 → 生成研判；数据缺失显式"跳过：未找到 X"，不编造）。页面自动刷新进度；你可以关掉页面，完成后飞书单聊推送结果卡。提问标的范围 = 全体标的池并集 + 你的持仓，池外标的提示先加自选；研究请求若触发新的个股分析，该分析并入公共分析库的正常节奏发布（不即时暴露"谁在研究什么"）。
3. **研判页**（版式即 §3.5）：核心结论（一行观点 + 置信度三档）→ 关键要点 → 数据表 → 与你的论点/纪律的对照 → 证据链（每条可点）→ 调研过程（可展开）。
4. **归档**：研判自动进报告库（类型=研判，仅本人可见）；研判中的判断可一键"存为论点"。研判可设为公开（进名片的"公开研判"区）——公开前系统检查其中引用的②档论点/纪律并弹出确认："公开将连带暴露以下 N 条系统可用档内容，是否一并升为公开？"，不确认则对照块以折叠脱敏形式公开。

**边界**：每人每日 ≤10 次；单次调研预算与超时固定（超时降级为"已收集材料+未完成研判"如实呈现）；研判默认仅本人可见。

### 1.4 报告页

列表页：类型筛选片（日报/周报/个股分析/研判/复盘）+ 日期倒序卡片。**日报/周报/个股分析是公共的；研判与复盘默认仅本人的可见。**

阅读页（吸收 FinSight 版式，全类型统一骨架）：**摘要卡先行**（核心结论+置信度）→ 可折叠目录（滚动高亮当前章节）→ 正文 → 来源清单（每条可点）→（研判类）调研过程可展开。数据表格横向滚动；图表静态渲染、关键数值标注在图上。

**个人页**：日报/周报各配"我的个人页"（登录后从报告页进入）：我的持仓速览、我的策略对照（"系统可用+公开"档策略与论点 vs 当日行情，距失效线距离）、我的提醒回顾。**个人页只有本人可见**——"系统可用"档策略绝不泄露给其他成员。

### 1.5 新闻页

与 07-11 版一致：事件聚类一事一卡（多源合并、来源+时间戳、影响标签、原文链接），顶部按标的/主题筛选（筛选范围=全体成员标的池并集），可翻近 7 天。

### 1.6 模拟盘页

**默认显示自己的盘**，顶部成员切换（看对方=只看其选择公开的部分，见 §1.8 名片）＋**对比视图**（两人净值曲线同图、同期基准参考线——良性竞赛）。对方关闭战绩展示时：对比视图只画本人曲线+基准并如实标注"对方未公开战绩"，首页对比入口同步置灰。

区块（本人视图）：KPI 行（净值/今日/累计/最大回撤）→ 净值曲线（vs QQQ 基准+回撤标注）→ 持仓表（横向滚动）→ 持仓当日涨跌条形图 → 仓位分布环图 → 提案与成交历史（五态徽章；点击进提案详情页）。

### 1.7 策略页

三段式：**我的纪律**（置顶：规则+执行方式徽章[代码强制/提案检查/自我约束]+近 30 天遵守情况）→ **我的策略卡与论点**（含私有档在本地、此处显示"系统可用+公开"两档，档位随时可调）→ **圈子公开区**（其他成员的公开策略/论点，按名片分组）。

论点卡沿用：看多/看空双栏 + 目标区间 + 失效条件 + 判断历史时间线（append-only，代码回算标注事后走势）。

### 1.9 下钻页与登录页

**个股页**（`/stock/<code>`）：头部（代码/名称/数据时间）→ 最新公共分析摘要卡 → **我的论点卡**（登录后，本人的；其他成员公开论点并列显示）→ 我的该标的提醒历史 → 历史分析列表。
**提案详情页**：状态条 → 提案原文 → 纪律检查 → 引用证据 → 审批与执行时间线 → 事后表现。仅本人可见自己的提案详情；对方提案对你可见的只有其选择公开的战绩汇总。
**登录页**：邮箱 → 6 位验证码 → 30 天会话；非白名单邮箱得到与正常发送相同的模糊提示。

### 1.10 平台不做什么

实时行情/K 线/站内下单；站内评论/点赞/关注/私信；公开注册；全市场选股器（只研究自己的池子，不扫 5000 只票——FinSight 调研的明确不抄项）；重型多 agent 辩论剧场（借鉴其过程透明 UI，不搬 agent 群）。

## 2. 策略记忆（三档可见性 + 双层存储）

### 2.1 三档可见性（每条策略/论点独立设置，默认②）

| 档 | 存哪 | 谁能用 | 谁能看 |
|---|---|---|---|
| ① 私有 | 只在你本地工作台 | 只有你本地的 OpenClaw | 只有你 |
| ② 系统可用 | 平台（按 owner 隔离） | 平台为**你**生成报告/提案/研判时注入 | 只有你 |
| ③ 公开 | 平台 | 同上 + 进你的名片 | 圈内所有成员 |

一键升档（①→②→③）；降档时已生成的历史内容不回收（如实告知）。

### 2.2 记录类型（沿用 07-11 版六类，全部加 owner）

纪律规则（结构化字段在交易数据库做硬检查，全文镜像进记忆库）、策略卡、个股论点（结构化数值入库供代码回算）、判断批注、交易批注（审批卡顺手采集）、复盘结论。字段定义与 07-11 版一致。**摩擦控制**：任何时刻人工必填 ≤2 字段。

### 2.3 采集与注入

采集：审批卡顺手采集（主路径）/ 飞书一句话 / 读报告反馈 / 本地工作台整理（经 skill 按档位发布）/ 月度复盘沉淀。
注入（都只用**本人**的"系统可用+公开"档）：个股分析的"我的策略对照"个人块、日报个人页、提案生成时的纪律检查与策略引用、站内研判的论点对照、月度复盘。
降级：记忆层不可用只影响叙述性上下文；**纪律硬检查读交易数据库，不受影响**。

### 2.4 防自欺与留痕（沿用）

违反纪律必填；论点验证与策略胜率由代码回算（样本 <10 标注"样本不足"）；判断历史 append-only；更正=新记录替代+旧记录作废标记；纪律可停用不可删历史；公开的论点自动带事后走势（公开即接受检验）。

## 3. 报告版式

### 3.1 公共日报（每美股交易日开盘前，北京时间约 20:00）

摘要卡（核心结论+置信度）→ 市场速览（大盘/板块）→ 多源新闻（事件聚类，覆盖全体标的池并集）→ 今日要点分析（引用编号证据）→ 事件深挖（top 2-3，每事件 ≤5 轮）→ 宏观与财报日历 → 来源清单。**不含任何个人持仓与策略内容。**

### 3.2 个人页（每人一份，随日报生成）

我的持仓速览（净值/涨跌/异动标注）→ 我的策略对照（论点距失效线、相关纪律提示）→ 我的提醒回顾 → 我的待办。发本人飞书单聊摘要卡；平台上仅本人可见。

### 3.3 周报

公共周报（周度行情归因、周事件聚类、深挖 3-5 个每事件 ≤8 轮且须含反方证据或明示未找到、下周前瞻）+ 每人个人页（本周我的交易 vs 策略一致性回顾）。

### 3.4 个股分析（公共资产；每 3 天批量 + 按需 + 站内研究触发）

9 段结构沿用 07-11 版（报价技术面/估值/基本面/分析师与情绪/期权链只读/新闻事件/多路径推演/结论框[置信度三档]/策略对照）。**策略对照段个人化**：正文 1-8 段公共；第 9 段为登录者动态渲染"我的论点 vs 本次分析"（无论点显示引导）。前 8 段 ≥6 段有事实表支撑，数字经代码比对。

### 3.5 研判页（站内研究的产物，FinSight 版式吸收落点）

核心结论（一行观点+置信度三档+"截至"时间）→ 关键要点（3-5 条，每条带证据角标）→ 数据表（涉及标的行情/关键指标）→ **与我的论点/纪律对照**（一致/冲突显式标注）→ 建议动作（如止损位/仓位建议，明确标注"不构成投资建议，模拟盘语境"）→ 证据链 → 调研过程（可展开的步骤流水，含跳过项）。

### 3.6 月度复盘（每人一份，每月第一个周末）

我的预测复盘（命中率+置信度校准）、我的决策复盘（提案收益 vs 拿住不动；被拒提案简化口径）、我的策略与纪律复盘（按策略统计+遵守率+"守规矩值多少钱"对比）、我的提醒质量、改进建议（变更须本人确认）。发本人单聊；平台仅本人可见；确认后结论入个人记忆。可选生成一页**圈子公共月报**（两人公开战绩对比+公开论点命中榜——都同意才开）。

## 4. 飞书交互

**单聊（个人通道）**：提醒卡、提案审批卡、个人页摘要、月度复盘、研判完成通知。审批卡只认本人点击；提醒规则/标的池/策略记忆的修改只认本人（`只有 owner 能动自己的东西`是所有写操作的统一规则）。
**群（公共通道）**：公共报告发布卡、系统告警、日常讨论、@ 机器人对话（每人都能用全部对话能力，操作自动落在自己名下）。
**对话能力**：分析请求、提醒规则 CRUD（自己的）、标的池维护（自己的）、记忆写入（自己的，含档位指定："公开记一条策略…"）、提案请求（自己的盘）、误报反馈、状态查询。
**提醒四类规则**与频控沿用 07-11 版（阈值/滞回/冷却/合并/每股每类型 ≤10 条；规则标的必须 ∈ 自己的标的池 ∪ 自己的持仓），配额改为**每人**每日 ≤30 张，一周误报 ≤3 张/人的验收门槛沿用。
**提案审批卡**沿用 07-11 版 §4.2 全部条款（收盘后自动+按需触发、每次 0-2 条且 0 条合法、卡片字段与批准/减半批准/拒绝三按钮、秒级回执、重复点击幂等、24h 无操作作废并回改卡片、纪律检查印在卡上、"无已批准提案不可成交"的服务端强制、熔断周亏 >3% 停一周且跨重启有效），仅三处变化：发到各成员本人单聊、各批各的盘、熔断按成员独立计算。

## 5. 接入与多机

### 5.1 mini（平台服务器）

自动化生产（报告/提醒/提案/研判执行）+ Web 应用 + 平台接口（查询/发布/研究任务，token 认证）+ 各成员数据隔离存储 + 唯一飞书连接。重启自愈；同类失败 3 次停机告警等复位。

### 5.2 成员的本地 OpenClaw（个人工作台）

装 **AlphaLoop 客户端 skill** + 个人 token 即接入：本地自治（私有策略/偏好在本地）；skill 提供的能力——查平台数据（行情/分析/新闻/自己的持仓提案）、读写自己的策略记忆（按档位）、发布/升档策略论点、触发研究任务、**管理自己的提醒规则与请求提案**（与飞书对话能力对齐，都只作用于自己名下）。本地关机不影响平台任何功能。

### 5.3 新成员接入流程（30 分钟目标）

①你在 mini 上执行添加成员命令（邮箱+飞书账号+生成 token）→ ②对方开长桥账号（模拟盘）并提供凭据给平台托管（或暂不开盘、先用其余功能）→ ③对方本地 OpenClaw 装 skill 填 token → ④进飞书群 + 收到欢迎单聊 → ⑤平台白名单生效，登录即见自己的空间。验收：按文档操作 30 分钟内完成全流程。

### 5.4 不分布的东西

飞书连接、各成员模拟盘执行与交易账本、平台记忆库、定时任务——全部只在 mini。成员本地只有私有层与客户端。

## 6. 运行与自愈

沿用 07-11 版全部条款（点火含 mini 环境盘点、凭据重建+模板+密钥清单、交易库与记忆库每日备份 30 天+上线恢复演练、宪法文档同步、同类失败 3 次停机待复位、降级透明、每周报告人工抽查），新增：
- **成员与 token 管理**：添加/吊销成员命令；token 泄露一键吊销即时生效。
- **多账户凭据隔离**：每成员的长桥凭据独立存放、互不可见；下单服务按提案 owner 选用对应凭据。

## 7. 验收清单（全部实测导向）

**平台**：两个账号分别登录，各自首页/个人页/模拟盘/策略页内容正确隔离（A 看不到 B 的"系统可用"档与个人页——双账号实测取证）；**负向隔离实测**：B 用深链直开 A 的提案详情页与研判页被拒、报告列表页 B 看不到 A 的研判/复盘条目、B 关闭战绩展示后 A 侧名片与对比视图正确隐藏；本人深链全部可达；页面无外部网络请求（浏览器工具核验）；未登录/白名单外进不来；家庭 IP 不出现在 DNS；重启自愈。
**站内研究**：提问→过程页实时步骤→研判页归档→飞书通知，全链路实测 3 个问题（含一次数据缺失场景显示"跳过"而非编造）；每日配额生效；离开页面后结果不丢。
**报告**：公共日报连续 5 交易日新闻达标（≥60% 非券商源、≥30% 中文源、链接抽查 ≥5 条全可达）；两人的个人页各自正确；数字比对坏样本全拦截；个股分析 3 只代表性标的（大盘股/高波动股/ETF）经你抽查认可；策略对照段按登录者正确渲染。
**双模拟盘**：两人各自完成"提案→批准→成交→留痕"与"拒绝""超时作废"全链路；A 无法批 B 的提案（实测）；无审批记录的下单请求被服务端拒绝；各自熔断独立触发（历史回放验证）。
**提醒**：各自规则触发/滞回/冷却/配额单测+实测各一次；互相改不了对方规则（实测）。
**策略记忆与名片**：三档可见性各实测一条端到端（私有不出本地→系统可用进自己报告不进对方视野→公开进名片）；升档/降档行为正确；判断历史不可删（实测）。
**接入**：mashu 按 §5.3 文档从零接入 ≤30 分钟（真实计时）；token 吊销即时生效；**B 的 token 调平台接口读 A 的持仓/提案/记忆被拒（越权负向实测）**。
**复盘**：两人首月复盘各自生成，指标经独立脚本回算一致。

## 8. 明确不做清单

实盘/期权自动化（宪法）；实时行情与站内下单；公开注册与商业化；站内社交（评论/点赞/关注/私信）；全市场选股器；多 agent 辩论剧场；共享模拟盘（各玩各的盘）；站内操作类指令（规则/审批走飞书）。

## 附录 B：FinSight 竞品结论（摘要）

该产品未在公网检索到发行痕迹（独立开发者项目）；可借鉴：结论先行+显式置信度、调研过程透明侧栏（含"数据缺失即跳过"的诚实降级）、观察池中心导航、BYO 模型优雅降级；明确不抄：全市场选股器、多 agent 辩论剧场、面向公众的多租户打磨。定位差异被调研确认：它与同类开源项目全部是**无状态问答工具**，我们的壁垒是**有状态**（持仓/论点/判断历史/纪律全在系统里）——借其交互外壳，垫我们的记忆内核。
