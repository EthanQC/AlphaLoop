## 0. 系统总览

**一句话**：一套跑在 Mac mini 上、你和 mashu 两个人每天用的美股交易辅助系统——它自动产报告、盯盘提醒、提案给你批、记住你们的策略和判断，并且越用越懂你们。

### 0.1 角色与设备

| 角色/设备 | 职责 |
|---|---|
| **Mac mini**（无人值守服务器） | 跑全部自动化：行情抓取、报告生成、提醒轮询、提案生成、模拟盘执行、看盘站点、策略记忆库。全年无休，重启自愈 |
| **你**（owner，唯一审批人） | 批提案、管提醒规则、定策略与纪律、月度复盘拍板 |
| **mashu**（协作者） | 看站点全部内容、群里跟机器人对话、写自己的策略卡和个股论点；**不能**批提案、改你的提醒规则 |
| **你的 MacBook / mashu 的 MacBook**（个人工作台） | 各自跑本地 OpenClaw 做深度工作（策略整理、复盘批注、研究笔记），成果沉淀进共享的策略记忆库 |
| **飞书** | 三件事：推通知、对话、一键审批。所有正文都在站点，飞书只是铃铛和对讲机 |

### 0.2 七类交互全景

| # | 交互 | 发生在哪 |
|---|---|---|
| 1 | 读报告/分析/复盘 | 看盘站点（报告页） |
| 2 | 读新闻 | 看盘站点（新闻页） |
| 3 | 看状态（持仓/规则/提案历史） | 看盘站点（模拟盘页、首页） |
| 4 | 收通知（异动/提案/故障） | 飞书卡片（深链跳站点） |
| 5 | 说一句话（要分析/管规则/记判断） | 飞书对话；MacBook 本地 OpenClaw |
| 6 | 批提案 | 飞书卡片按钮 |
| 7 | **策略记忆**（策略/纪律/论点的沉淀与回流） | 采集：飞书+审批卡+MacBook；呈现：站点策略页；回流：自动注入报告与提案 |

### 0.3 立项原则（全文有效）

- **不做实时行情看盘**——那是长桥/众安 App 的事，本系统不抢。站点数据按分钟到小时级刷新，且每个数字都标注数据时间。
- **时间约定**：全站与报告时间戳统一显示北京时间；日报在每个美股交易日**开盘前**（北京时间约 20:00）生成，覆盖上一交易时段收盘至当时的信息；页面上的"提醒流水"指**最近一个美股交易时段**（北京时间约 21:30 至次日凌晨 4/5 点）；提醒的"每日一次/每日配额"按美东交易日切界。
- **标的池**：你们通过机器人维护的关注清单（上限 20 只）是全系统对"关注什么"的唯一定义——分析、新闻、提醒都以"标的池 + 当前持仓"为范围。
- **预算与频控**（沿用既定数值）：新闻检索日报 ≤30 次、周报 ≤60 次、事件深挖每事件 ≤5 轮（周报 ≤8 轮）；报告生成总时长 ≤15 分钟；提醒卡全局每日 ≤30 张。
- **PDF 已退役**：一切正文在站点，飞书只发摘要卡；历史 PDF 保留，仓库中旧的"发 PDF"口径文档随点火一并修订。
- **不分步、一次做齐**（你已明确）：四类提醒、日报事件深挖、每日自动提案、三类图表全部随首版上线。
- **机械数据自动填、人只填判断**（策略日志调研的核心教训：80% 的日志因录入摩擦在两个月内被弃用）。
- **"不确定"是一等公民**：报告与提案必须带证据与不确定性标注；系统给判断材料，不替人判断。
- 沿用既定安全宪法（永不实盘、期权永不自动化、只有下单服务可触达券商、模拟盘 ≤10% 预算服务端校验、凭据不入仓不入提示词不入报告、飞书白名单、中文）。

## 1. 看盘站点（Web 平台）

### 1.1 全局设计

**访问**：`https://reports.<你的域名>`。白名单邮箱验证码登录（v1 = 你 + mashu 两个邮箱），登录会话 30 天。未登录一律跳登录页。

**导航**：固定底部标签栏，5 个入口（图标 + 中文短标签）：

```
┌─────────────────────────────────────┐
│              页面内容                │
│                                     │
├─────────────────────────────────────┤
│  🏠首页  📄报告  📰新闻  💼模拟盘  🎯策略 │
└─────────────────────────────────────┘
```

依据：5 个一级页面正好是底部标签栏的适用上限；移动端单手可达；每个 tab 有稳定 URL，飞书卡片深链直接落到对应页且标签栏仍在（可横向切换）。

**每页通用元素**：
1. 页顶**生成时间条**：`生成于 07-11 21:05 · 数据截至 21:00`＋新鲜度标签（`最新`/`延迟`/`部分缺失`）。数据过期或缺失时显示降级横幅（如 `⚠ 行情源延迟，以下数字截至 20:30`），变灰对应区块而不是隐藏——诚实优先。
2. 全部中文、移动优先（单列卡片流，重要内容在首屏）；桌面浏览器下内容区居中限宽。
3. 页面完全自包含：不向任何第三方发起网络请求；图表为静态图形，关键数值直接标注在图上（实现方式见技术选型文档 §2.3）。体验目标：手机已登录状态下点开数秒内可读——是体验目标，不作验收门槛。
4. 深链规则稳定：`/daily/2026-07-11.html`、`/stock/NVDA.html`、`/proposal/P-123.html` 等，供飞书卡片引用，永不失效。

**一句话**：打开就知道"今天发生了什么、有没有需要我处理的"。

```
┌─────────────────────────────────────┐
│ 07-11 周五 · 生成于 21:05 · [最新]    │
├─────────────────────────────────────┤
│ 模拟盘概览                            │
│ 净值 1.083  今日 +0.6%  累计 +8.3%    │
│ ▁▂▃▅▄▆▇ (30日净值 sparkline)         │
├─────────────────────────────────────┤
│ ⏳ 待处理                             │
│ ▸ 提案 P-124 NVDA 买入 2 股 (23h后失效)│
├─────────────────────────────────────┤
│ 🔔 最近交易时段提醒 (3)                │
│ 22:10 NVDA 日内 -4.3%                │
│ 23:05 TSLA 浮亏 -6.2%                │
│ 00:40 组合敞口 10.4% 超限             │
├─────────────────────────────────────┤
│ 📄 今日日报                           │
│ 「联储纪要偏鹰，半导体走弱…」          │
│ [阅读全文 →]                          │
├─────────────────────────────────────┤
│ 🎯 纪律速览：财报周不加仓 (NVDA 7/28)  │
└─────────────────────────────────────┘
```

**区块自上而下**（顺序即优先级：待办与时效信息在上，报告是压轴）：

| 区块 | 内容 | 数据来源 | 交互 |
|---|---|---|---|
| 模拟盘概览条 | 净值（期初=1.0）、今日/累计收益、30 日净值 sparkline | 小时级账户快照 | 点击 → 模拟盘页 |
| 待处理 | 未决提案卡（标的/方向/数量/剩余有效期） | 提案表 | 点击 → 提案详情页（含"去飞书批"提示；审批动作只在飞书） |
| 提醒流水 | 最近一个美股交易时段的全部提醒（时间/内容/类型徽章） | 提醒事件表 | 点击 → 对应个股页 |
| 今日日报卡 | 标题+两行摘要 | 日报产物 | 点击 → 报告阅读页 |
| 纪律速览 | 与近期事件匹配的 1-2 条活跃纪律。匹配规则：持仓临近财报→财报类纪律；仓位接近上限→仓位纪律；周亏接近 3%→熔断预警；无匹配→显示"纪律已连续遵守 N 天"默认态 | 纪律规则 | 点击 → 策略页 |

无事发生时各区块显示友好空态（"今日无提醒"），不隐藏区块。

### 1.3 报告页

**列表页**：顶部类型筛选片（全部/日报/周报/个股分析/复盘），下面按日期倒序的报告卡（标题、类型徽章、日期、一行摘要）。筛选在页面内即时生效（轻量实现，见技术选型文档 §2.3）。

**阅读页**（长文阅读体验是本页的全部意义）：

```
┌─────────────────────────────────────┐
│ 日报 · 2026-07-11                    │
│ 生成 21:05 · 行情截至 21:00 · [最新]  │
├─────────────────────────────────────┤
│ ▍摘要卡（先给结论）                    │
│  今日要点 3 条 + 持仓影响一句话         │
├─────────────────────────────────────┤
│ ▸ 目录（可折叠，当前章节高亮）          │
├─────────────────────────────────────┤
│ 正文（章节见 §3 各报告版式）            │
│  · 数据表格横向滚动，不撑破页面         │
│  · 图表内联 SVG，标注直接画在图上       │
│  · 新闻条目带 [1][2] 角标 → 文末来源    │
├─────────────────────────────────────┤
│ 来源清单（每条新闻/数字的原文链接）      │
│ [回到顶部]                            │
└─────────────────────────────────────┘
```

要点：开头必有**摘要卡**（先结论后细节，Seeking Alpha 的 thesis-first 模式）；目录做成正文内可折叠块并随滚动高亮当前章节；中文排版（合理字号/行距/段距，参考微信公众号阅读密度）；所有引用可点击跳到来源清单。

### 1.4 新闻页

**一句话**：你们标的池+持仓的当日新闻，按"事件"聚类，一件事一张卡，不刷屏。

```
┌─────────────────────────────────────┐
│ 筛选: [全部|NVDA|TSLA|宏观|...]       │
├─────────────────────────────────────┤
│ ▍事件卡：联储 6 月纪要偏鹰             │
│  影响: 利空成长股 · 涉及: 全部持仓      │
│  摘要两行…                            │
│  来源: 财联社 21:02 / WSJ 20:45 /     │
│        华尔街见闻 21:10   [展开 5 条]  │
├─────────────────────────────────────┤
│ ▍事件卡：NVDA 财报日确认 7/28          │
│  …                                   │
└─────────────────────────────────────┘
```

- 同一事件的多源报道**合并成一张卡**（复用管线的去重聚类），卡内列出全部来源与各自时间戳——不出现二十条近似重复。
- 每条来源显示媒体名+相对时间（点击显示绝对时间），链接到原文。
- 顶部按标的/主题筛选。当日新闻之外可翻近 7 天。
- 每张事件卡带"影响"标签（方向/涉及标的/理由），来自新闻引擎的结构化输出。

### 1.5 模拟盘页

**一句话**：模拟盘的完整账本——净值、持仓、每一笔提案与成交的历史，全部可回溯。

区块自上而下：
1. **KPI 行**：净值、今日收益、累计收益、最大回撤（四个大数字）。
2. **净值曲线**（SVG 折线）：模拟盘净值 vs 同期 QQQ 基准参考线，最大回撤区间浅色标注，关键点（大额成交日）打点。
3. **持仓表**（横向滚动）：代码/名称/数量/成本/现价/市值/仓位占比/浮动盈亏/当日涨跌/贡献度。持仓集中度一目了然。
4. **持仓涨跌条形图**：各持仓当日涨跌的横向条形（涨跌红绿分明，值标在条上）。
5. **仓位分布环图**：按标的占比，含现金。
6. **提案与成交历史**（时间倒序列表）：每条 = 日期 / 标的方向数量 / 状态徽章 / 一句话理由 / 关联策略标签。状态徽章五态：`待审批`(黄)/`已批准·已成交`(绿)/`已拒绝`(灰)/`已过期`(灰)/`熔断暂停`(红)。点击进提案详情页（版式见 §1.7）。

### 1.6 策略页（新功能的门面）

**一句话**：你们俩的交易大脑外置——纪律、策略、每只股票的论点与判断历史，全部在这一页，而且它们真的在影响系统的每一次分析和提案。

```
┌─────────────────────────────────────┐
│ ▍交易纪律（置顶，永远可见）             │
│  ☑ 单标的仓位 ≤10%      [代码强制]     │
│  ☑ 财报周不开新仓        [提案检查]     │
│  ☑ 周亏 >3% 停一周      [代码强制]     │
│  ☐ 不追高开盘 30 分钟    [自我约束]     │
├─────────────────────────────────────┤
│ ▍策略卡                               │
│  「AI 算力龙头持有」 abble · 活跃       │
│   适用: NVDA/AVGO · 3 笔关联交易       │
│  「超跌反弹短线」 mashu · 暂停          │
├─────────────────────────────────────┤
│ ▍个股论点                             │
│  NVDA · 看多 · abble · 07-02 更新     │
│  ┌─────────────┬─────────────┐       │
│  │ 看多依据 3 条 │ 风险/看空 2 条│       │
│  └─────────────┴─────────────┘       │
│  目标区间 850-950 · 失效条件: 跌破 720  │
│  判断历史 ▾                           │
│   07-02 上调目标（财报预期）→ 现价 +6%  │
│   06-15 初始论点 → 之后 -3% 触及支撑    │
└─────────────────────────────────────┘
```

三种卡片：

1. **纪律规则卡**（置顶）：规则文本 + 执行方式徽章——`代码强制`（系统硬性执行，如 10% 仓位）/`提案检查`（每条提案生成时自动核对并在审批卡标注"符合/违反"）/`自我约束`（仅提醒）。每条规则显示近 30 天的遵守情况（如"3 次提案全部符合"）。
2. **策略卡**：名称、owner（你/mashu）、状态（活跃/暂停/退役）、适用场景、进出条件要点、关联交易数与简单胜负统计（**样本小于 10 笔时明示"样本不足，不作结论"**——防小样本假因果）。
3. **个股论点卡**：标的、方向（看多/看空/中性）、owner、核心逻辑的**看多/看空双栏**（Morningstar Bulls Say/Bears Say 模式）、目标区间、失效条件、更新时间；下方**判断历史时间线**——每次判断记录当时的话+之后的实际走势（由代码回算标注，如"→ 现价 +6%"），新事件对照旧论点（"财报后论点是否仍成立"）。**论点是否被验证由确定性代码算（价格 vs 目标/失效线），不由 AI 宣称**。

### 1.7 下钻页与登录页

**个股页**（`/stock/{代码}.html`，报告 tab 的下钻页；提醒卡与提醒流水的落地页）：

```
┌─────────────────────────────────────┐
│ NVDA 英伟达 · 数据截至 21:00          │
├─────────────────────────────────────┤
│ ▍最新分析摘要卡（结论框要点）           │
│  [读完整分析 →]                       │
├─────────────────────────────────────┤
│ ▍你们的论点（有则显示，无则空态）        │
│  看多 · abble · 目标 850-950          │
├─────────────────────────────────────┤
│ ▍该标的提醒历史（近 30 天）             │
├─────────────────────────────────────┤
│ ▍历史分析列表（倒序）                  │
└─────────────────────────────────────┘
```

区块自上而下：头部（代码/名称/数据时间）→ 最新一篇个股分析的结论框摘要卡（链接全文）→ 该标的论点卡（复用策略页样式）→ 近 30 天提醒历史 → 历史分析列表。该标的无分析时显示空态与"在飞书说『分析一下 XX』"的引导。

**提案详情页**（`/proposal/{编号}.html`）：区块顺序 = 状态条（五态徽章+关键时间）→ 提案原文（标的/方向/数量/理由/失效条件/止损/预算占比/置信度）→ 纪律检查结果（逐条 ✓/✗）→ 引用证据（事实与新闻链接）→ 审批与执行时间线（谁、何时、动作）→ 事后表现（复盘回填前显示"待复盘"）。

**登录页**：输入邮箱 → 收 6 位验证码 → 登录 30 天。非白名单邮箱显示与"验证码已发送"相同的模糊提示（不暴露白名单成员关系）；验证码时效与重发间隔按托管方默认。

### 1.8 站点不做什么

- 不做实时行情、K 线、下单（看盘用券商 App；审批只在飞书）。
- 不做站内评论/聊天（讨论在飞书群）。
- 不做账号体系（白名单邮箱验证码即全部）。
- v1 不做站内写入操作（策略卡/论点的创建编辑走飞书对话或 MacBook 工作台，见 §2.2）；站点是只读呈现。

## 2. 策略记忆（核心新功能）

### 2.1 五种记录类型（字段级定义）

| 类型 | 字段 | 谁写 | 存哪 |
|---|---|---|---|
| **纪律规则** | 规则文本、执行方式（代码强制/提案检查/自我约束）、关联策略、启用状态、owner | 人（飞书/工作台） | **结构化字段在交易数据库**（提案纪律检查读这里，硬保证不依赖记忆层）；全文镜像进策略记忆库供检索 |
| **策略卡** | 名称、适用场景、进场条件、风控（止损/仓位）、出场规则、状态、owner | 人 | 策略记忆库 |
| **个股论点** | 标的、方向、看多依据[]、看空/风险[]、目标区间、失效条件、owner、更新时间 | 人 + 系统草拟（个股分析的结论框可一键"存为论点"） | **结构化数值（目标区间/失效条件）在交易数据库**（供代码回算验证）；全文在策略记忆库 |
| **判断批注** | 追加到某论点判断历史的一条批注：文本、关联报告/提案、owner、时间 | 人（读报告/复盘时随手说） | 策略记忆库（挂在对应论点的判断历史时间线上） |
| **交易批注** | 关联提案、所用策略、一句话理由、纪律检查结果、事后教训（可空） | 审批时自动采集 + 人补充 | 结构化字段在交易数据库（提案表），文本镜像进策略记忆库 |
| **复盘结论** | 周期、覆盖交易、纪律遵守率、错误归类、一句话教训、下一步动作 | 月度复盘生成 + 人确认 | 人确认后写入策略记忆库 + 方法论笔记 |

**摩擦控制（硬要求）**：任何时刻要求人手动填写的必填字段 ≤2 个。机械数据（价格、盈亏、时间、检查结果）全部系统自动填。

### 2.2 采集路径（怎么写进去）

1. **审批卡顺手采集**（主路径，零额外动作）：提案卡上自带"关联策略"（系统按标的和内容预选，可改）与提案理由；你点批准的同时这笔交易的批注就自动成型。事后想补一句，群里说`批注 P-124 财报前最后一次加仓`即可。
2. **飞书一句话**：`记一条纪律：财报周不开新仓`、`NVDA 论点：看多，目标 900，跌破 720 失效`、`把超跌反弹策略暂停`——机器人解析成结构化记录，回卡片确认后入库。
3. **读报告时的反馈**：你在群里对某份报告说`我觉得这个目标价太乐观`，机器人问一句"追加为 NVDA 论点的判断批注？"，确认即挂到该论点的判断历史上（类型定义见 §2.1）。
4. **MacBook 工作台**（深度整理）：你或 mashu 在各自 MacBook 的本地 OpenClaw 里做策略研究、复盘长文，让本地 agent 把结论提炼后写入共享策略记忆库（见 §5）。
5. **月度复盘沉淀**：复盘报告的"本月教训"，经你确认后自动入库。

所有记录带 owner（你/mashu）与来源标签，可追溯是谁在什么场景写的。

### 2.3 注入路径（怎么流回来）——这是"读完不再结束"的关键

| 场景 | 注入内容 | 呈现 |
|---|---|---|
| **个股分析生成时** | 该标的的现有论点+判断历史+相关策略卡 | 报告新增第 9 段「策略对照」：上次你怎么判断→之后实际走势→本次分析与论点的一致/冲突点。冲突时显式标注"本次分析与你 07-02 的论点方向相反，差异在于…" |
| **日报/周报生成时** | 活跃纪律中与当日事件相关的条目、临近失效条件的论点 | 报告"策略上下文"小节：如"NVDA 距你设的失效线 720 还有 4.2%""下周三 NVDA 财报，纪律：财报周不开新仓" |
| **提案生成时** | 全部活跃纪律 + 相关策略卡 + 该标的论点 | 提案必须声明引用哪条策略；纪律逐条自动核对，结果印在审批卡上（`✓ 符合仓位纪律 / ✗ 违反财报周规则`）；**违反"代码强制"级纪律的提案直接不生成** |
| **月度复盘时** | 全部策略卡与纪律 | 按策略统计收益与遵守率；论点按代码回算标注验证状态 |

**降级语义**：策略记忆服务不可用时，报告照常生成、只是没有策略上下文块，并在报告头标注"策略记忆不可用"——绝不因记忆层故障阻塞任何管线。**纪律硬检查不受此影响**：纪律的结构化规则存在交易数据库（§2.1），提案管线直接读它，因此"违反代码强制纪律的提案不生成"在记忆服务停机时依然成立；记忆层降级只影响叙述性的策略上下文。

### 2.4 防自欺设计

- 交易批注中"违反了哪条纪律"是**必填**（无违反则自动填"无"）——防止只记好的不记坏的。
- 论点验证、策略胜率全部由代码从价格数据回算，AI 只做叙述；样本 <10 笔的统计必须带"样本不足"标注。
- 两人互相可见全部策略与论点（互相监督），但只能编辑自己的（owner 校验）。

### 2.5 修正与留痕语义（防"清洗历史"）

- **判断历史 append-only**：论点可以更新、可以标记"撤回"，但历史条目永不删除——表现差的判断不能被抹掉，这正是复盘的原料。
- **纪律规则可停用、不可删历史**：停用的规则保留全部遵守/违反记录。
- **写错了怎么办**：更正走"新记录替代 + 旧记录标记作废"（作废记录不再注入报告，但在站点历史中可见）。
- 所有变更留 owner 与时间痕，站点判断历史时间线如实展示。

## 3. 报告版式（每种报告具体长什么样）

通用规则：全中文；每个数字可溯源（来自事实表，AI 叙事中的数字经代码比对，误差超限自动重写 ≤2 次、仍失败降级为纯事实表报告并标注）；每份报告头部含生成时间与各数据源截止时间；美股节假日不产报告。

### 3.1 日报（每美股交易日开盘前生成，北京时间约 20:00）

| # | 章节 | 内容规格 |
|---|---|---|
| 0 | 摘要卡 | 3 条今日要点 + 一句话持仓影响（AI 写，引用有据） |
| 1 | 模拟盘速览 | 净值/今日收益/持仓涨跌表（前 5+其余合并）+ 30 日净值 sparkline |
| 2 | 持仓与标的池行情 | 表格：现价/涨跌/成交量对比均量；异常行高亮（当日触发过提醒的标注 🔔） |
| 3 | 多源新闻（事件聚类） | 每事件：标题（中文）/影响（方向/标的/理由）/两行摘要/来源角标；≥60% 非券商源、≥30% 中文源；每条带可点原文链接 |
| 4 | 今日要点分析 | AI 撰写的中文分析段，只允许引用本报告事实表与新闻编号；含不确定性表述 |
| 5 | 事件深挖 | 当日 top 2-3 事件的多源交叉核实：事件-证据-影响-不确定性（每事件 ≤5 轮检索） |
| 6 | 宏观日历 | 未来 14 天重要宏观事件+持仓财报日 |
| 7 | 策略上下文 | 见 §2.3：纪律相关提示、论点距失效线距离；无相关内容时显示"今日无策略提示" |
| 8 | 执行与提醒回顾 | 当日成交（如有）、提醒流水、提案状态变化 |
| 9 | 来源清单 | 全部链接 |

### 3.2 周报（每周日晚）

日报结构基础上：①周期改为一周（净值周曲线、周度涨跌归因——哪只贡献最大）；②新闻按周聚类只留重要事件；③事件深挖扩展至 3-5 个、每事件 ≤8 轮检索，且每个事件须包含至少一条与主流叙事相反的证据（找不到则明确写"未找到反方证据"）；④新增"下周前瞻"（财报/宏观/技术位）；⑤策略上下文扩展为"本周策略回顾"（本周交易 vs 策略的一致性）。

### 3.3 个股分析（每 3 天批量 + 按需）

沿用 8 段模板并升级，另新增第 9 段「策略对照」：

| # | 章节 | 升级点 |
|---|---|---|
| 1 | 报价与技术面 | 现价/均线/趋势评分（现有） |
| 2 | 估值 | PE/PB + 同自身历史分位（不做同业分级——已确认砍） |
| 3 | 基本面 | **新增**：营收/毛利/自由现金流增速（数据源见技术选型文档 §2.8） |
| 4 | 分析师与市场情绪 | **新增**：分析师目标价共识、空头仓位（更新频率低，数据时点随值标注） |
| 5 | 期权链只读观察 | 现有，仅分析用 |
| 6 | 新闻与事件 | 该标的近期事件聚类 + 财报日 |
| 7 | 多路径推演 | 现有 3 路径概率 |
| 8 | 结论框（结构化） | 竞争力判断/合理价值区间+依据/**置信度三档**/当前价格位置/复盘触发条件与日期；因子强弱标签（自身数据启发式） |
| 9 | **策略对照**（新增） | 你的现有论点 vs 本次分析：一致/冲突点；上次判断的事后走势；一键"更新论点"入口（飞书） |

机构持仓（13F）不做（数据滞后 4.5 个月，已确认砍）；前 8 段中 ≥6 段有事实表数据支撑（第 9 段策略对照不计入该统计），缺数据段落显式标注原因；结论写入预测库供复盘。

### 3.4 月度复盘（每月第一个周末）

1. **预测复盘**：到期预测 vs 实际，方向命中率、三档置信度校准（代码回算）。
2. **决策复盘**：已执行提案收益 vs 同期持有不动基准；被拒提案的假想收益（提案价→复盘日价的简化口径+固定免责说明）。
3. **策略与纪律复盘**（新增）：按策略统计交易与盈亏（小样本标注）；纪律遵守率；违反纪律的交易 vs 遵守纪律的交易的收益对比（TradeZella 的"守规矩值多少钱"报告）。
4. **提醒质量**：触发数/误报数（来自你的反馈）/防刷屏机制统计。
5. **本月改进建议**：阈值/提示词/新闻源/策略调整建议——建议 only，变更须你确认。
6. 每项指标可由独立脚本从原始记录回算比对（防复盘报告自己编数）；你确认的结论沉淀进策略记忆库与方法论笔记。

## 4. 飞书交互

### 4.1 提醒卡

四类规则一次上线（阈值可各自调整）：

| 类型 | 默认阈值 | 频率 | 防误报 |
|---|---|---|---|
| 持仓日内涨跌 | ±4% | 每日一次 | — |
| 持仓浮动盈亏 | ±6% | 持续（滞回 1%） | 相对成本价 |
| 5 分钟急涨急跌 | ±2.5% | 持续（冷却 60 分钟） | 连续 3 周期有成交 |
| 敞口超预算 | >10% | 持续 | 服务端口径一致 |

卡片内容：`⚠ NVDA 日内 -4.3%（阈值 ±4%）· 持仓 12 股，影响 -$520 · [查看详情→站点个股页]`。中文、含当前值/阈值/影响金额/深链。

频控（防刷屏）：单规则同方向 60 分钟冷却；滞回复位；同窗口合并成一张卡；每股每类型 ≤10 条规则；全局每日 ≤30 张（按美东交易日重置）；发送遵守飞书同群限速。美股交易时段内持续监测，触发到送达 ≤5 分钟（监测机制见技术选型文档 §2.6）。

规则管理：群里自然语言（`NVDA 跌破 800 提醒我`/`把特斯拉的提醒关了`/`提醒规则列表`）。**只有你能增删改**，mashu 只能查看。规则标的必须 ∈ 标的池 ∪ 持仓。误报反馈：对着提醒卡说`这条没用`即标记，进月度复盘统计。

### 4.2 提案审批卡

- 触发：每交易日收盘后自动 + 群里说`给我提案`按需。每次 0-2 条，**0 条是常见合法输出**。
- 卡片（发到与你的**单聊**，不进群——审批通道只属于唯一审批人，避免群内误点与噪音；提案与持仓在站点上对两位用户本就可见）：

```
📋 提案 P-124 · 买入 NVDA 2 股 · 限价 $845
理由: 财报前动量+论点目标未到 [引用: 事实F3, 新闻N2, 论点T-NVDA]
关联策略: AI 算力龙头持有 ▾(可改)
纪律检查: ✓ 仓位 8.2%<10%  ✓ 非财报周  ✓ 未触熔断
失效条件: 跌破 $720 · 止损: $760 · 置信度: 中
预算影响: 占模拟盘预算 6.8%
[ 批准 ] [ 减半批准 ] [ 拒绝 ]     23:58 后自动作废
```

- 点击秒级回执，异步执行；**只认你本人点击**；重复点击不重复下单；24h 无操作自动作废并回改卡片；成交后卡片更新为成交回报+站点链接。
- **纪律检查印在卡上**：`提案检查`级逐条核对显示 ✓/✗；`代码强制`级违反的提案根本不会生成。
- 提案通知只发给你；mashu 可随时在站点看到全部提案与状态。他若也想收到提案的推送通知（只读、无按钮），后续可开，v1 不做。
- **"未批准不可成交"是系统级保证**：下单服务只接受携带已批准提案编号的指令（服务端校验），包括你手动下单也走"生成提案→自批→执行"同一链路，全程留痕。
- 熔断：模拟盘周亏 >3% 自动暂停新提案一周并通知；暂停状态跨重启有效。

## 5. 多机协作（每台机器的体验）

### 5.1 Mac mini（无人值守服务器）

跑全部自动化与共享服务：交易栈、报告管线、提醒监测、看盘站点、**共享策略记忆库**（专用实例，与个人记忆隔离）、唯一的飞书连接——**群内对话的主 agent 常驻 mini**（它就是飞书连接的持有者）。重启全自愈；故障飞书告警；同类失败 3 次停机等你复位（不会悄悄无限重试）。

### 5.2 你的 MacBook（个人工作台）

- 日常轻交互全走飞书+站点，**不需要开电脑**。
- 深度工作时：本地 OpenClaw 照常做你的个人事务；做交易策略研究时，它可以**读**共享策略记忆（查你和 mashu 的论点、策略、判断历史）并把整理好的结论**写**回去（经确认）。
- 需要交易数据时（如让本地 agent 分析持仓），通过受控通道从 mini 只读获取——凭 token、可随时吊销。
- 你的 MacBook 关机不影响系统任何功能。

### 5.3 mashu 的 MacBook

与你的工作台能力一致（本地 OpenClaw + 读写共享策略记忆 + 只读交易数据），权限差异只有两条：不能审批提案、不能改你的提醒规则。他写的策略/论点带他的 owner 标签，与你的并列呈现在策略页。

### 5.4 协作模型（产品语义）

- **单一事实源在 mini**：交易账本（数据库）、策略记忆库、报告产物。MacBook 上不保存权威副本，工作台是"读-想-写回"。
- 两人对**同一个**论点意见不同时：各写各的论点（同一标的允许两条论点并存，owner 不同），策略页并列展示——系统不裁决观点分歧，把分歧可视化。
- mini 离线时：站点打不开、提醒停发、工作台读写记忆失败（有明确报错），恢复后自动续上。你接受此单点风险（已确认）。

## 6. 运行与自愈（用户可见行为）

1. **点火**（开发机→mini 部署）：R0 第一步是**mini 环境盘点**（现状未知，已确认开工时实测）；随后凭据重建（提交无真实值的模板+密钥清单文档）、全套安装、备份就位、两项冒烟测试（带按钮的测试卡点击链路、群 @ 回复）。验收含一次重启全自愈实测。
