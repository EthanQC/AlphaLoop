01

环境盘点

迁移后运行时全停，先确认 mini 上到底有什么、缺什么。别假设任何东西还在。

需人工

确认 mini 基础环境

Node ≥24（仓库 node:sqlite 依赖 24+）、pnpm 9.15.0、git、可访问 git@github.com:EthanQC/AlphaLoop。缺 Node 先装。

node -v # 需 v24+ pnpm -v # 9.15.0 git clone git@github.com:EthanQC/AlphaLoop.git && cd AlphaLoop && pnpm i && pnpm build
就绪

跑一次 doctor 看现状

此刻大概率满屏红（网关/runner 未监听、launchd 未装、RSSHub 不可达）——这是预期基线，逐阶段消红。doctor 本身崩溃已在 P2.5 修好，不会因单项失败而不打印。

pnpm openclaw:runtime:doctor
02

凭据重建

迁移丢了 .env.local。按 docs/superpowers/specs/secrets-inventory.md 逐项重填——凭据不入仓，只能你手动放。

阻断

重建 .env.local

拷 .env.local.example 起步。必填：LONGBRIDGE_APP_KEY/SECRET/ACCESS_TOKEN、LONGBRIDGE_ACCOUNT_MODE=paper、ALLOW_LIVE_EXECUTION=false（宪法永久禁实盘）、飞书 LARK_*。P4/P7 新增：FINNHUB_API_KEY、RSSHUB_BASE_URL、MEMORYD_DATA_ROOT。P6 新增：BROKER_EXECUTOR_SHARED_SECRET（执行器缺它拒绝启动，这是先记录后执行链的第一道门）。

前置于 P1 / P4 / P6 / P7
阻断

渲染 ~/.openclaw/openclaw.json

render-openclaw-config.mjs 把 .env.local 里的飞书 FEISHU_APP_ID/SECRET、白名单、requireMention、网关 OPENCLAW_GATEWAY_TOKEN 落盘到 ~/.openclaw/openclaw.json——没有任何安装脚本会自动跑它，必须手动执行一次（改任何密钥后都要重渲染并重启相关服务）。所需密钥全部来自你、永不入仓；OPENCLAW_GATEWAY_TOKEN 未固定时每次重渲染都会生成新 token，已配对客户端全失效。

node apps/openclaw-config/scripts/render-openclaw-config.mjs
前置于 P3 网关 / 飞书接线
需人工

每成员独立券商凭据目录

P6 的多账户隔离要求 ~/.alphaloop/credentials/<member-id>/longbridge.env，权限 0600，只有 broker-executor 可读，子进程 env 注入、永不进全局 process.env。你和 mashu 各一份（mashu 未开户则先留空 = 无盘成员降级）。

前置于 P6 多账户
需人工

飞书 oauth 写回自检

P4 审计修过 oauth 写 .env.local 会毁引号/注释的 bug。跑完 oauth 后 git diff 确认只改了目标 key、其余逐字节不变。

05

外部依赖

四个外部件：长桥 CLI、RSSHub 容器、Finnhub key、memoryd 专用实例。都是本地代码替代不了的真实服务。

需人工

长桥 CLI + 模拟盘账户

装 longbridge 到 ~/.local/bin（env LONGBRIDGE_CLI_PATH 可覆盖）。P5 实测过：无 CLI 时个股分析管线诚实 fail-loud 而非假装成功——所以这一步缺了不会静默出坏数据，但真数据得靠它。第二个（mashu）真实账户单列。

前置于 P5 / P6
需人工

RSSHub Docker 容器

P4 的中文新闻源（财联社/华尔街见闻/格隆汇）走本机 RSSHub，仅绑回环。launchd 模板已就位（com.alphaloop.rsshub，docker start 包装），但容器本体要你先建。三条中文路由历史上常失效——P4 已配第二冗余路由，但真实连通性得点火后验。

docker run -d --name rsshub -p 127.0.0.1:1200:1200 diygod/rsshub
前置于 P4 新闻引擎
需人工

Finnhub key + memoryd 专用实例

Finnhub 免费 60 次/分，填 FINNHUB_API_KEY（缺则该源标 skipped_no_key，不报错）。memoryd 起一个 MEMORYD_DATA_ROOT 专用实例、仅绑回环（HTTP 无鉴权=回环即信任边界）、每成员一个 scope。策略记忆的镜像层此前全是 fire-and-forget 降级，接上后才真正写入。

前置于 P4 / P7
06

可注入后端接线

全程贯穿的设计：任何依赖真实运行时的能力都做成可注入后端，真实现是「P10 抛错占位」，测试注入 fake。本阶段把五个占位换成真的。

阻断

五个 create*Backend() 占位实现

① createResearchBackend()（P8）→ OpenClaw 受限 agent 网关（仅搜索抓取、无 shell、owner-scope 预绑定）+ 搜索配额实测；② createOpenclawSearchBackend()（P4，news-agent-search.mjs）→ 新闻的受限 agent 搜索后端，与研判后端相互独立、同样是 P10 抛错占位；③ createNarrativeLlmBackend()（P5）→ headless LLM 运行时；④ createMemorydBackend()（P7）→ memoryd 回环 HTTP；⑤ 飞书结果卡投递 createFeishuReviewNotifier()（P8/P9）。每个都在源码里标了接线点，且在没接上时优雅降级——研判会显式跳过、叙事回落纯事实表、镜像标 mirrored:false，都不崩。

前置于 P4 / P5 / P7 / P8 / P9

飞书接线

卡片按钮审批（ocf1）走 OpenClaw 原生 card.action.trigger，无需新回调栈。但三个真实配置得你在飞书后台开。

阻断

开通 im:message 权限

P8/P9 实测时真实撞到 99991672 Access denied——应用尚未开通 im:message:send / im:message / im:message:send_as_bot 任一。不开，提醒卡/审批卡/研究结果卡/复盘卡全发不出去。

P8 点火时实撞
需人工

订阅 card.action.trigger + requireMention 冒烟

P2.5 评审确认 OpenClaw 2026.5.28 原生把按钮点击转成合成消息注入 control agent（自带 token 去重/过期/绑定点击者）。两个残余验证点：飞书后台是否已订阅该事件；requireMention:true 是否拦截合成消息。任一失败 → 回落文本审批（同管道，按钮 value=文本命令）。

08

Cloudflare Access（身份网关）

公网入站零；两个子域经 cloudflared 出站隧道全指向 platform-app（:4314），由它做唯一身份执法点。

阻断

verifyAccessJwt 必须变成真的

P3 现在是文档化 no-op（返回 true），仅靠 loopback-only 兜底安全。真实部署必须校验 Cf-Access-Jwt-Assertion 对 AlphaLoop 真实 Cloudflare 团队域的 JWKS——否则任何本地进程都能伪造 Cf-Access-Authenticated-User-Email 头冒充身份。源码里 TODO(P10) 标了接线点。

前置于 P3 身份网关
需人工

cloudflared 双子域 + Access 白名单

reports.<域名>（Cloudflare Access 邮箱白名单）+ api.<域名>（edge Access service token + app 层 bearer，双层）。两个子域都指向 platform-app；静态服务 :4313 永不直接暴露。白名单=你+mashu 邮箱。

前置于 P3 平台
09

服务安装

launchd 装齐：platform-app、market-alerts、daily-backup、rsshub、official-paper、reports/stock-analysis（用户级）；gateway 与 broker-executor 是系统级守护进程，另由 launchd:install-system 单装。另有独立的 cron-runner 常驻进程带 5 个 OpenClaw cron 任务（含 monthly-review），再单独装。P2.5 已统一用户级 launchd 安装脚本，别只跑一条。

就绪

两条安装命令都要跑

P2.5 查明 launchd:install-user（报告/选股/official-paper）和 install-launchd.sh（备份/提醒/网关）装的是不重叠的两套——只跑一条会装出半套系统。README 已并排文档化两条命令。P4 的 rsshub launchd 模板已接入 glob。

pnpm launchd:install-user pnpm launchd:install-backup-alerts
前置于 P2.5 接线
阻断

装 cron-runner 常驻进程 + 5 个 cron 任务

openclaw:cron:install 是 com.openclaw.trading.cron-runner 这个 launchd 常驻进程的唯一安装器：装好后一次性注册全部 5 个 OpenClaw cron 任务——daily-report、weekly-report、stock-analysis、proposal-sweep、monthly-review。monthly-review 是跑在 cron-runner 下的 cron 任务（pnpm reviews:generate），不是 launchd 模板。doctor 的 runner-listeners 硬要求这个进程在 18792 端口上单实例监听，缺了就报红、系统起不来。

pnpm openclaw:cron:install
前置于 P4 / P9 cron-runner
阻断

装系统级守护进程（gateway + broker-executor）

launchd:install-system（=install-system-daemons.sh）把 ai.openclaw.system.gateway 和 com.openclaw.system.trading.broker-executor 装成 root 拥有的系统级 launchd 守护进程——不是用户级、也不归 cron-runner 管。需 sudo。broker-executor 缺 BROKER_EXECUTOR_SHARED_SECRET 会拒绝启动；缺了它下单链路整条不通。

sudo pnpm launchd:install-system
前置于 P6 执行器 / P3 网关
就绪

doctor 应转绿

装完 doctor 应报 launchd 任务已 load、alerts-poller 心跳新鲜、ALERTER-DOWN.json 不存在、rsshub 健康。心跳/失败升级是 P2.5 连修五轮的核心——「静默死掉的提醒器」是这套系统最坏的失败模式，doctor 是唯一的外部观察者。

前置于 P2.5 可观测性
10

冒烟测试

只能线上验的项——本地单测覆盖了导出函数，但真实运行时的行情供给、飞书投递、下单只有点火后才摸得到。

需人工

美股盘中跑 alerts:poll --dry-run

P2 遗留的明确义务：defaultQuoteProvider 从未在真实行情下跑过。必须在美股交易时段实跑一次 dry-run，确认真拿到行情、真评估、真组卡——之后才能信任它无人值守运行。

pnpm alerts:poll -- --dry-run # 美股盘中
前置于 P2 提醒引擎
需人工

真实飞书投递 + 卡片按钮往返 + 真实下单

给自己发一张真提醒卡确认落地；点一次审批卡按钮确认 ocf1 合成消息注入管线；真跑一次先记录后执行的下单（模拟盘），确认 lifecycle 落库→下单→回执→对账全序，且重试幂等不重复下单。

前置于 P2 / P6

需人工

隔离矩阵（真实 A/B）
